Das Verzeichnis ~krypto/protokoll/local/

server.sh Bevor ein Protokoll ausgeführt werden kann, muß ein Server gestartet werden, der die Kommunikation zwischen den teilnehmenden Parteien (Clients) gewährleistet. Da dieser Server mehrere Protokolle gleichzeitig verwalten kann, braucht nur ein Server pro Host gestartet zu werden.
server.sh ist ein einfaches Shellscript, welches den Server mit dem erforderlichen vollständigen Klassennamen startet.
client.sh Eine .class-Datei, die den Ablauf eines Protokolls beschreibt, wird ausgeführt, indem jede teilnehmende Partei einen Client startet und in einem der sich dann öffnenden Eingabefenstern den Namen dieser .class-Datei auswählt.
client.sh ist ein einfaches Shellscript, welches den Client mit dem erforderlichen vollständigen Klassennamen startet.
Skeleton.java Hierin sind die leeren Methoden sendFirst und receiveFirst mit den für das jeweilige Protokoll spezifischen Algorithmen auszufüllen. Die Klasse Communicator stellt hierfür Methoden zur Kommunikation mit den anderen Parteien zur Verfügung.
example.java Ein Beispielprotokoll, das als nicht wörtlich zu nehmendes Beispiel dienen mag, wie eigene Protokolle anzugehen sind.
